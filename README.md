# Plane MCP-сервер для self-hosted Plane

Кастомный MCP-сервер, адаптированный для self-hosted версии Plane. Позволяет управлять задачами, проектами и страницами в Plane прямо из Claude Code или Cursor.

---

## Самый простой способ (если у вас есть Claude Code / Cursor)

> Если не хотите разбираться — просто скопируйте эту папку в свой проект и напишите в чат Claude:
>
> **"Настрой мне подключение к Plane через MCP"**
>
> Claude прочитает инструкции из этой папки и проведёт вас через настройку — спросит API-ключ, slug и URL, сам создаст конфигурацию. Вам нужно будет только ответить на его вопросы и перезапустить IDE.

---

## Ручная настройка (пошагово)

### 1. Установите UV

**Windows (PowerShell):**

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

**macOS / Linux:**

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

Перезапустите терминал и проверьте: `uv --version`

### 2. Получите API-ключ Plane

1. Откройте Plane в браузере → нажмите на **иконку профиля** в правом верхнем углу → **Настройки**

   ![Шаг 1 — Откройте настройки профиля](images/step1-profile-settings.png)

2. Вы попадёте на страницу настроек. Переключитесь на вкладку **Account**

   ![Шаг 2 — Страница настроек Workspace](images/step1b-workspace-settings.png)

3. В левом меню в разделе **Developer** выберите **Personal Access Tokens** → нажмите **Добавить токен**

   ![Шаг 3 — Раздел API-токенов](images/step2-api-tokens.png)

4. Введите название токена (например, `Cursor`), включите **Бессрочный** → нажмите **Сгенерировать токен**

   ![Шаг 4 — Создание токена](images/step3-create-token.png)

5. Скопируйте ключ (начинается с `plane_api_`) и **сохраните его** — он показывается только один раз!

   ![Шаг 5 — Копирование токена](images/step4-copy-token.png)

> **Важно:** ключ показывается только один раз. Сохраните его сразу.

### 3. Определите Workspace Slug

Workspace Slug — короткий идентификатор рабочего пространства.

**Из URL:** откройте Plane → перейдите в workspace → посмотрите URL:

```
https://plane.akfixdev.ru/it/projects/...
                             ^^
                             это slug
```

**Из God Mode (для админов):** перейдите в `/god-mode/workspace/` — slug указан в квадратных скобках, например **IT / [it]**.

### 4. Скопируйте эту папку к себе

Скопируйте всю папку `plane-mcp-selfhosted` на свой компьютер и запомните полный путь, например:

```
C:\Users\ваше_имя\plane-mcp-selfhosted
```

### 5. Создайте `.mcp.json`

Создайте файл `.mcp.json` **в корне того проекта**, который вы открываете в VS Code / Cursor.

```json
{
  "mcpServers": {
    "plane": {
      "type": "stdio",
      "command": "uv",
      "args": ["run", "--directory", "ПУТЬ_К_ПАПКЕ", "python", "server.py"],
      "env": {
        "PLANE_API_KEY": "plane_api_ваш_ключ",
        "PLANE_WORKSPACE_SLUG": "it",
        "PLANE_BASE_URL": "https://plane.akfixdev.ru"
      }
    }
  }
}
```

**Замените:**

| Параметр | Что подставить |
|----------|---------------|
| `ПУТЬ_К_ПАПКЕ` | Полный путь к папке `plane-mcp-selfhosted` |
| `plane_api_ваш_ключ` | Ваш API-ключ из шага 2 |
| `it` | Ваш workspace slug из шага 3 |
| `https://plane.akfixdev.ru` | URL вашего Plane |

> **Windows:** если `uv` не в PATH, укажите полный путь:
> ```
> "command": "C:\\Users\\ваше_имя\\.local\\bin\\uv.exe"
> ```

### 6. Проверьте подключение (до перезапуска IDE)

Убедитесь, что всё работает, прежде чем перезапускать IDE:

**Windows:**

```powershell
$env:PLANE_API_KEY="plane_api_ваш_ключ"
$env:PLANE_WORKSPACE_SLUG="it"
$env:PLANE_BASE_URL="https://plane.akfixdev.ru"
uv run --directory ПУТЬ_К_ПАПКЕ python check.py
```

**macOS / Linux:**

```bash
PLANE_API_KEY="plane_api_ваш_ключ" \
PLANE_WORKSPACE_SLUG="it" \
PLANE_BASE_URL="https://plane.akfixdev.ru" \
uv run --directory ПУТЬ_К_ПАПКЕ python check.py
```

Если всё ок, увидите:

```
Подключение успешно! Найдено проектов: 1
  - IT [IT] (id: 018dd7d9-...)
```

### 7. Перезапустите IDE

Сохраните `.mcp.json` → полностью перезапустите VS Code / Cursor → откройте проект. MCP-сервер запустится автоматически.

### 8. Готово!

Напишите в чат Claude:

- *"Покажи список проектов в Plane"*
- *"Какие задачи в проекте IT?"*
- *"Создай задачу в бэклог: Тестовая задача"*

## Дополнительно: сессионные cookie

Некоторые функции (страницы, `get_me`) требуют cookie-авторизацию. Это **не обязательно** — без cookie будут работать все основные функции (задачи, проекты, статусы, метки, участники, циклы, модули).

### Когда нужны cookie

Только если вы хотите **создавать и редактировать страницы (Pages)** через AI-ассистента.

### Как получить

1. Откройте Plane в **Chrome**
2. Нажмите **F12** → вкладка **Application** (Приложение)
3. Слева: **Cookies** → выберите домен Plane
4. Скопируйте значения двух cookie:

| Cookie | Что это |
|--------|---------|
| `session-id` | Длинная строка — токен сессии |
| `csrftoken` | Короткая строка — CSRF-токен |

### Добавьте в `.mcp.json`

В блок `env` добавьте две строки:

```json
"PLANE_SESSION_COOKIE": "значение_session-id",
"PLANE_CSRF_TOKEN": "значение_csrftoken"
```

> **Важно:** cookie живет ~7 дней. Когда истечет — повторите шаги и обновите значения.

## Возможности

### Через API-ключ (работают всегда)

| Команда | Описание |
|---------|----------|
| `list_projects` | Список проектов |
| `get_project` | Детали проекта |
| `list_issues` | Задачи (с фильтрами по ответственному, статусу, приоритету, метке) |
| `create_issue` | Создать задачу |
| `update_issue` | Обновить задачу |
| `delete_issue` | Удалить задачу |
| `list_states` | Статусы проекта |
| `list_labels` | Метки |
| `create_label` | Создать метку |
| `list_members` | Участники проекта |
| `list_cycles` | Циклы |
| `list_modules` | Модули |

### Через session cookie

| Команда | Описание |
|---------|----------|
| `list_pages` | Список страниц |
| `create_page` | Создать страницу |
| `update_page` | Обновить страницу |
| `get_page` | Получить страницу |
| `get_me` | Информация о текущем пользователе |

## Решение проблем

| Проблема | Решение |
|----------|---------|
| MCP не запускается | Проверьте путь к `uv` и к папке. Запустите вручную: `uv run --directory ПУТЬ python server.py` |
| 401 Unauthorized | API-ключ неверный или истек — пересоздайте |
| 404 Not Found | Workspace slug не совпадает с URL, или лишний `/` в конце `PLANE_BASE_URL` |
| Pages не работают | Нужны session cookie (см. раздел выше) |
| Cookie истекли | Повторите шаги получения cookie и обновите `.mcp.json` |
